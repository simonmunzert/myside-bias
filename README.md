# Replication materials for: Myside Bias Shapes Public Preferences for Online Content Moderation

## Abstract

Online content moderation shapes what billions of people can say and see, and the human judgments that underpin it—how ordinary people perceive toxic speech and what they believe should be done about it—are deeply contested. Drawing on a pre-registered cross-national conjoint experiment with 19,172 respondents across 11 countries, we analyze over 150,000 lay judgments of toxic social media posts. We find robust evidence of myside bias: people are substantially more likely to classify speech as hateful and to support sanctions against it when the attacked target shares their position on the contested issue at hand. This pattern holds across countries, political camps, topics, and speech types, and persists when we control for respondents' general ideological alignment with the target, showing that myside bias is not reducible to partisan or ideological bias; by contrast, we find no consistent in-group favoritism based on gender or ethnic identity under the conservative, avatar-based operationalization used here. Because this bias is tied to the content of beliefs rather than to fixed personal characteristics, it cannot be neutralized by adjusting for stable demographic or partisan attributes. This poses a challenge for the many moderation approaches that rest on aggregating lay human judgment—from crowd annotation and user reporting to community-based systems that seek cross-ideological consensus—and raises the question, which our design cannot settle, of whether the same bias shapes professional moderators and the automated classifiers trained on human labels.

```

  cooked/  cleaned data used by the analysis scripts
```

## Reproducing the analysis

All analyses are run from R. Open the project root as the working directory and run:

```r
source("code/main.R")
```

`code/main.R` sources the scripts below in order. It first loads packages,
helper objects, and functions, then runs the descriptive, hypothesis-testing,
and revision-stage scripts, writing all figures and tables to `results/`.

### Script guide (run in this order)

| Script | Purpose |
|---|---|
| `packages.R` | Installs/loads required R packages. |
| `functions.R` | Project-specific helper functions (model tidying, confidence intervals via `add_confints()`, `gt` table builders, significance underlining). |
| `helper-objects.R` | Variable containers: outcome list (`outcome_vars`), covariate/label tables, country codes. |
| `01-import-data.R` | Imports raw survey data. |
| `02a-process-covariates.R` | Builds respondent-level covariates (ideology, gender, ethnicity/avatar, empathy, silencing, exposure, etc.). |
| `02b-process-vignettes.R` | Builds the long respondent-by-vignette data, including issue-stance alignment (`target_resp_issue_agreement`), ideology match (`target_resp_ideology_match`), and identity-match indicators. Saves `data/cooked/data_survey_combined.RData`. |
| `03a`–`03d` | Descriptive statistics: sample, covariates, vignettes, and outcomes. |
| `04a-analyze-main-vignettes.R` | Main vignette models (AMCEs, marginal means, respondent-level models, avatar models). |
| `04b-analyze-hypotheses-1-3-ingroup-outgroup.R` | Pre-registered H1 (myside bias) and H2–H3 (identity overlap), including Figure 4 (with the revision horse-race column). |
| `04c-analyze-hypotheses-4-5-framing.R` | Pre-registered H4–H5 (framing experiment). |
| `04d-analyze-hypotheses-6-exposure.R` | Pre-registered H6 (exposure / question-order experiment). |
| `05-analyze-reviewer-checks.R` | Revision robustness checks: within-person (Mundlak) identification, myside-vs-ideology horse-race, ad-type robustness, identity-bias-by-topic-relevance. |
| `06-subgroup-omnibus.R` | Revision analyses: Benjamini–Hochberg adjustment, omnibus subgroup interaction tests (Leeper et al. 2020), cross-national sorting and cross-pressure, task-order/carryover, country fixed-effects, treatment-arm invariance, and identity interaction tests. |

### Notes

- Models are hierarchical linear probability models (`lme4::lmer`) with random
  intercepts for respondents, vignette decks, and (in pooled analyses) countries.
- Plots use the **Fira Sans** font; install it locally or change the font family
  in the scripts if it is unavailable.
- `webshot_png_export` in `main.R` controls whether `gt` tables are exported as
  PNG (requires Chrome) or HTML.
- Significance is displayed at p < 0.01 (revision); see SI for the full list of
  deviations from the pre-analysis plan.
